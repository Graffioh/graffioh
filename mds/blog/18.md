# Empower a curl request: super-curl

Let's call it *super-curl* (or coincisely, **sc**).

I've been testing this since I started with my new job (~2 weeks) and it's honestly so good (and so simple).

It leverages your coding agent capabilities to send a curl request, automatically generates UUIDs and other parameters from the input, supports debugging and inspection through a `.txt` file.

For example, I use it to send multimodal generation (image, video and so on) requests to our main API endpoint with an input prompt.

You should set-up different sc-* commands/skills.

In my case I have `sc-send-generation`, it instructs the agent how to perform a POST request to a specific messages endpoint. To invoke it, I would've written a prompt like this:

> use the sc-send-generation skill with prompt: "Generate an image of a cat" and later give me the object storage link of the generated image

This will automatically generate relevant params such as: UUIDs, type of generation and others. But most importantly it will provide debugging & inspection capabilities.

If an error occurs, it will be in his context, it will be used to inspect the code and provide possible fixes iteratively.

Cool, right?

When invoked via the `/sc-send-generation` command, the user provides a generation prompt in the arguments.

## Execution

Run the `generate-and-send.js` script with the prompt from the user and the generation mode:

\```bash
bun .claude/skills/sc-send-generation/generate-and-send.js "<EXACT_USER_PROMPT>" "<GENERATION_MODE>" [chat_id] ...
\```

Where:
- `<EXACT_USER_PROMPT>` is the verbatim prompt from the user's command arguments
- `<GENERATION_MODE>` is "image" (default), "video", or other supported modes
- `[chat_id]` is optional, use "null" to skip and generate a new one
...

**Examples:**
\```bash
# Generate an image (with output saved to file for inspection)
bun generate-and-send.js "a cyberpunk cityscape at night" image 2>&1 | tee /tmp/generation-output-<timestamp>.txt

# Generate a video
bun generate-and-send.js "a chicken dancing" video 2>&1 | tee /tmp/generation-output-<timestamp>.txt

# Continue conversation with existing chat_id
bun generate-and-send.js "another prompt" image <chat-id> 2>&1 | tee /tmp/generation-output-<timestamp>.txt
\```

**Note:** The `2>&1 | tee /tmp/generation-output.txt` pattern captures both stdout and stderr, displays them in real-time, and saves them to a file for later inspection.
