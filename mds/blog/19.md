# How to write an argmax kernel

## Hierarchy

This hierarchy is the basis for interacting with the GPU (either via CUDA or HIP)

```blog-diagram
{
  "file": "cuda-execution-hierarchy"
}
```

It's important to memorize this layout to correctly utilize and optimize parallel usage with GPU memory - As opposed to the simple loops we would write when working on typical programming with CPU and RAM, here we must take into account every component in the hierarchy above.

The kernel function signature will be prepended with `__global__` to signal that it is a kernel function.

The main components given by the GPU API to interact with the hierarchy above are:

- `blockIdx` = Block inside the grid
- `blockDim` = Dimension of a block
- `threadIdx` = Thread in the block

Then it's possible to get *warps* and *lanes* (threads) with:

- `blockDim / WARP_SIZE` = Num of warps
- `threadIdx / WARP_SIZE` = Warp ID
- `threadIdx % WARP_SIZE` = Lane ID (thread's position in a warp)

The GPU assigns the index value to these components automatically during the parallel execution - e.g. `blockIdx` gets assigned a block index automatically while executing but we must think of interacting with all of the blocks simultaneously.

## argmax

Calculating an argmax at its core is a really simple operation: find the index associated to the max value inside a list of values.

Well, an optimized GPU kernel is not that straightforward at first, but I decided to write this blog so we can go through it together and understand the basics and the approach.

I'll annotate the code with comments and a diagram - for me this is the best way to learn with code instead of big paragraphs of explanation followed by just code.

## kernel

```cpp source=https://github.com/Luce-Org/lucebox/blob/main/server/deps/llama.cpp/ggml/src/ggml-cuda/argmax.cu#L90 title="kernel function call"
/*kernel*/argmax_f32<<</*grid=*/blocks_num, /*block=*/blocks_dim, 0, stream>>>(/*input=*/src0_d, /*output=*/dst_d, /*columns=*/ne00);
```

```cpp source=https://github.com/Luce-Org/lucebox/blob/main/server/deps/llama.cpp/ggml/src/ggml-cuda/argmax.cu#L9-L13 title="init"
  // the block viewed as a row
  const int64_t row = blockIdx.x; 
  // pointer to the row (row-major flatten layout)
  const float * rowx = row * ncols + x;

  // setup temp vars for argmax calculation
  float maxval = -INFINITY;
  int   argmax = -1;

  // explained in the previous section
  const int n_warps = blockDim.x / WARP_SIZE;
  const int lane_id = threadIdx.x % WARP_SIZE;
  const int warp_id = threadIdx.x / WARP_SIZE;
```

Now the core procedure is to perform an *all-reduce* across threads, warps and blocks so we'll get a single final tuple as a result.

The flow will be: 

- argmax across threads -> argmax across warps -> argmax across blocks -> final argmax

Let's start with finding argmax across the threads of a warp

```blog-diagram
{
  "file": "cuda-thread-level-argmax"
}
```

```cpp source=https://github.com/Luce-Org/lucebox/blob/main/server/deps/llama.cpp/ggml/src/ggml-cuda/argmax.cu#L15-L21 title="calculate thread-level argmax"
  // each thread gets blockDim distanced columns, optimized for parallel computation and locality
  for (int32_t col = threadIdx.x; col < ncols; col += blockDim.x) {
      // pick the value from the block row
      const float val = rowx[col];
    
      // calculate argmax
      if (val > maxval || (val == maxval && (argmax < 0 || col < argmax))) {
          maxval = val;
          argmax = col;
      }
  }
```

At the end, we'll have an argmax for each thread in a warp, but this is only the first level of the hierarchy.

Now we reduce the thread-level argmax values inside each warp so every (thread) lane obtains the same warp-level argmax:

```blog-diagram

  "file": "cuda-warp-level-argmax"
}
```

```cpp source=https://github.com/Luce-Org/lucebox/blob/main/server/deps/llama.cpp/ggml/src/ggml-cuda/argmax.cu#L23-L31 title="reduce thread-level results for warp-level argmax"
// macro for faster loop computation
#pragma unroll
    // reduce-split-compare warps each iteration: 
    // 16 -> 8 -> 4 -> 2 -> 1
    for (int offset = WARP_SIZE/2; offset > 0; offset >>= 1) {
        // compare the tuple between two warps
        const float val = __shfl_xor_sync(0xFFFFFFFF, maxval, offset, WARP_SIZE);
        const int   col = __shfl_xor_sync(0xFFFFFFFF, argmax, offset, WARP_SIZE);

        if (col >= 0 &&
            (val > maxval || (val == maxval && (argmax < 0 || col < argmax)))) {
            maxval = val;
            argmax = col;
        }
    }
```

What is `__shfl_xor_sync`??? 

It is used to compare stuff across warps using their respective registers (so we avoid extra shared memory movement and things are fast af) by selecting a *xor partner*: `lane_partner = lane_src ^ offset`

We can then compare all warp-level argmax to have an argmax across all warps:

```cpp source=https://github.com/Luce-Org/lucebox/blob/main/server/deps/llama.cpp/ggml/src/ggml-cuda/argmax.cu#L36-L62 title="retrieve argmax among all warps"
  // if we have only one warp, we already have the final argmax
  if (n_warps > 1) {
      // maximum number of warps in a block
      // NOTE: a block can contain at maximum 1024 threads
      constexpr int    max_warps = 1024 / WARP_SIZE;
    
      // shared memory tuple
      __shared__ float shared_maxval[max_warps];
      __shared__ int   shared_argmax[max_warps];

      // we already obtained a thread-level argmax previously, so we just need to pick it from the first lane (avoiding extra work)
      if (lane_id == 0) {
          shared_maxval[warp_id] = maxval;
          shared_argmax[warp_id] = argmax;
      }

      // wait until all warps populate the respective shared memory
      __syncthreads();

      // from the first warp (in each block), we store again the tuple in the register vars, to compute faster the block-level argmax
      if (warp_id == 0) {
          if (lane_id < n_warps) {
              maxval = shared_maxval[lane_id];
              argmax = shared_argmax[lane_id];
          }

// same reduction as before - this time we have warp-level argmax in the registers!
#pragma unroll
          for (int offset = WARP_SIZE/2; offset > 0; offset >>= 1) {
              const float val = __shfl_xor_sync(0xFFFFFFFF, maxval, offset, WARP_SIZE);
              const int   col = __shfl_xor_sync(0xFFFFFFFF, argmax, offset, WARP_SIZE);
              if (col >= 0 &&
                  (val > maxval || (val == maxval && (argmax < 0 || col < argmax)))) {
                  maxval = val;
                  argmax = col;
              }
          }
      }
  }
```

And now, let's save the final argmax (across all warps in the block) into the respective block destination:

```cpp source=https://github.com/Luce-Org/lucebox/blob/main/server/deps/llama.cpp/ggml/src/ggml-cuda/argmax.cu#L64-L66 title="save block-level argmax"
  if (warp_id == 0 && lane_id == 0) {
      dst[row] = argmax;
  }
```
